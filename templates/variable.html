This section describes a pattern to create variables names.

Since having an ontology of all existing variables is not likely to exist, we can have a pattern that helps create new variables.
This is also usefull to understand variables created by someone else, if we all use the same pattern.


Entity - Quality - (Method) - Unit


With :
Entity : being the scientific object the variables is focusing on.
    Exemples : soil, air, plant, leaf, CropCanopy, ...
Quality : being the scientific quality (or quantity) measured. This is described in the Ontology of Measuremenr (OM).
    Exemples : Ex: Temperature, Mass, Growth Degree Day, Vegetation Index, DimensionLess
Method : being the different methods/protocols you declared for experimentation (for naming a variable it is optional, mention if it's relevant)
    Exemples : Imaging computation, Hand measurement, Simulations
Unit : being the scientific unit described in the Ontology of Measurement.
    Exemples : Â°C, kg, cm.days-1, UnitLess

This pattern for naming variables will undoubtly lead to some variables being more difficult to name than others. Special cases:

  • In order to precise a subpart of your entity or method (like soil from 0 to 30cm deep or a tree at 2m height) you can alter the name (just the name) of the entity to create variables.
    This way takes into account constraints of measurement.
  • Variables that do not fit well in this model. Some variables (for example environmental indices) will be difficult to declare, Unit and Quality or even Entity will be hard to choose.
    For those variables, I suggest you to use blanks and we will see after collecting a number of them, if we can find a pattern.
