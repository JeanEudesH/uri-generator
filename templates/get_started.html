    3. Generate URI for your own installation.

    You need to create files that describes objects in your installation, for each category of objects you are interested in. They need to have the same format as in the corresponding object category in examples (ยง2 above).
    The file needs a first line with a header (name of colomns), followed by one row per individual (e.g. plant 1, plant 2, etc). Each line represents a particular object (e.g. one plant or one sensor), each column contains the required information for creating the URI. Each column contains variables applying to the considered objects (label, genotype, variety, etc). The expected format for the file is a "csv" with commas (,) as a delimiter between fields.

    Non conventionnal structure

    If your file contains comments at the beginning, for example "Data acquired on the 15/05/2020, by Jean-Michel Saumon" you can skip this row using the argument in the "Details" section.
    As a rule, do not insert metadata in the file itself, better insert them in a separate file and link the two files, this will avoid you a lot of problems
    The expected format for the table is the "csv" format, a classic format to manipulate data, this is very simple (just text) and use the comma (,) as a delimiter between fields. If you are not using the classic delimiter you can specify it in the "Details" section.

    Recover your URIs and check them

    You can check what was the last value given to your uri in the list URI menu. By default, the file with URIs at each line is stored in the 'Downloads files' folder of your computer, under the name "export_URI_leaf" (ear, plot, etc...).
    Because identifiers are meant to be persistent, it is impossible to delete an identifier once created. It is also not possible to reuse an identifier generated in the past for a resource that do not exist anymore. Hence, a new URI will be created if you submit a file twice. You can use any of the URIs created for the considered object.
    You can check what was the last value given to your uri in the list URI menu. Some resources are not incremental (data, images, ...) and thus are using a random URI generated.
