Get started

In this page we will see how to easily use this application with some simple examples.

Initialize account

First of all, you have to log in. A TEST user (test/test) is avaliable to perform some queries and get used to it.
But you also want to create your own user, for that, Create your account. it will set up a new database for you.

Test user

Before messing your database with unintended queries and create URI you don't want, you can go have a try with the test user.
Log in with the test/test details.
If you go to the list URI menu, you will realize that some URI have already been generated. It's time to generate your own.

Generate URI

    Data structure

    To make it simple, this software reads the dataset you uploaded, and creates a URI for each rows of this dataset according to the informations you filled in the required sections.
    This means that your table has to be "tidy". That is to say starting with a header row, followed by a row for each individuals and a column for each variables.

    Non conventionnal structure

    If your table has some comments at the beginning, for example "Data acquired on the 15/05/2020, by Jean-Michel Saumon" you can skip this row using the argument in the "Details" section.
    The expected format for the table is the "csv" format, a classic format to manipulate data, this is very simple (just text) and use the comma (,) as a delimiter between fields. If you are not using the classic delimiter you can specify it in the "Details" section.

    Warnings

    It is important to notice that if you submit the same dataset twice, it will give you new URI each time, increasing the incremental count of the selected resource.
    Because identifiers are meant to be persistent, it is impossible to delete an identifier once created. It is also not possible to reuse an identifier generated in the past for a resource that do not exist anymore.

    Check your records

    You can check what was the last value given to your uri in the list URI menu. Some resources are not incremental (data, images, ...) and thus are using a random URI generated cryptographically.
